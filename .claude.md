# CLAUDE.md — South African Financial Compliance App

## Project Overview

A Next.js 14 web application for assessing South African financial compliance and audit readiness. Companies upload trial balance data (CSV/Excel), and the system runs automated compliance checks against IFRS, Companies Act 71, and SARS tax rules, producing scored reports with PDF export.

## Tech Stack

- **Framework:** Next.js 14 (App Router) with TypeScript (strict mode)
- **Database:** Prisma ORM + SQLite (dev); schema at `prisma/schema.prisma`
- **Styling:** Tailwind CSS
- **PDF:** `@react-pdf/renderer`
- **File parsing:** `xlsx` (Excel), custom CSV parser at `src/lib/compliance-engine/parser.ts`
- **UI components:** Custom primitives in `components/ui/` (Button, Card, Badge)

## Architecture Notes

- All pages are client components (`'use client'`) using `useEffect` + `fetch` for data loading
- The compliance engine is pure TypeScript with no DB dependencies — checks receive parsed data and return results
- API routes handle Prisma operations and orchestrate the compliance engine
- The `cn()` utility exists in both `src/lib/utils.ts` and `components/ui/button.tsx` (duplication to fix)
- Database uses cascade deletes: deleting a company removes all its trial balances, reports, and check results

## Compliance Engine

10 checks in `src/lib/compliance-engine/checks.ts`:
1. **TB_BALANCE** — Trial balance debits = credits
2. **REV_RECOG** — Revenue recognition (IFRS 15)
3. **TAX_PAYE** — PAYE payable vs salary ratio
4. **TAX_UIF** — UIF contribution (1% of salaries)
5. **TAX_VAT** — VAT output > input verification
6. **COS_RATIO** — Cost of sales margin check
7. **DEPR** — Depreciation adequacy (PPE)
8. **RECV** — Trade receivables vs revenue ratio
9. **TAX_INC** — Income tax provision (27% rate)
10. **EQUITY** — Positive equity check (Companies Act s4)

Each returns a `ComplianceResult` with `ruleCode`, `status` (COMPLIANT/WARNING/HIGH_RISK), `score` (0-100), and `details`.

## Known Issues

See `review.md` for a comprehensive code review. Key items:
- `.env` is tracked in git (needs `git rm --cached`)
- No authentication/authorization
- Float used for financial amounts (should be Decimal)
- No tests
- Mass assignment vulnerability in PATCH `/api/companies/[id]`

## Conventions

- API routes return `NextResponse.json()` with appropriate HTTP status codes
- Error responses follow `{ error: string }` shape
- Financial amounts are in ZAR (South African Rand)
- Tax rates: PAYE ~19%, UIF 1%, VAT 15%, Corporate tax 27%
- UI uses Tailwind with emerald/slate color scheme
